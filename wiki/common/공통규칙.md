# SERO 공통 규칙

모든 화면 요구사항은 아래 규칙과 충돌 여부를 검증해야 한다.
규칙 변경은 아키텍처 협의체 승인 사항이며, 화면 단위 예외는 해당 화면 문서에 예외로 기록한다.

## 그리드 (GRID)
- [GRID-01] 모든 목록 그리드의 기본 페이지 사이즈는 **20건**이다.
- [GRID-02] 그리드 정렬은 **서버 사이드**로 처리한다. (클라이언트 정렬 금지)
- [GRID-03] 데이터 0건일 때 "조회된 데이터가 없습니다." 문구를 그리드 영역에 표시한다.

## 삭제 (DEL)
- [DEL-01] 데이터 삭제는 **논리 삭제**(사용여부='N')로 처리한다. 물리 삭제 금지.
- [DEL-02] 삭제 전 confirm 문구는 "선택한 항목을 삭제하시겠습니까?"로 통일한다.

## 사원 도메인 (EMP)
- [EMP-01] 사원번호는 시스템 자동 채번이며 **수정 불가** 항목이다.
- [EMP-02] 사원명은 최대 30자, 특수문자 입력 불가.

## 권한 (AUTH)
- [AUTH-01] 삭제 버튼은 **관리자 권한 그룹에만 노출**한다. (비노출, 비활성 아님)

## 검증 (VAL)
- [VAL-01] 필수 입력 누락 시 필드 하단 인라인 메시지 + 해당 필드 포커스로 처리한다. (alert 금지)
